Dev12 Gateway
work-in-progress

Wi-Fi connectivity for a RFM69 network or a Lora™ network.

  • ESP32-WROOM-32x Wi-Fi+BT+BLE module
  • RFM69HCW or RFM95/96/97/98(W) module
  • high efficiency synchronous buck converter
  • supply voltage supervisor
  • Li-Ion / Li-Polymer battery charger and socket for 18650 or 14500 backup battery
  • OLED display and 4 LEDs for status
  • placeholder for ATSHA204A, ATECC508A or ATECC608A IC
  • placeholder for a humidity and temperature sensor
Dev33 DLED
work-in-progress

It is designed to control either:

  • two strings of digital LEDS like WS2812, WS2812B, ... (aka NeoPixel) and derivatives like SK6812;
  • one string of APA102 (aka DotStar) and derivatives like SK9822.

Uses the ESP32's RMT peripheral and two real logic level shifter circuits.

It also controls the power supply of LED strings.

Dev34 DLEDGrid
beta

Born as a proof-of-concept, it is a Dev33-DLED with an 8x8 grid of WS2812 LEDs running in Soft AP mode and controlled through a web interface.

Uses Simple OTA with firmware upload from a HTTP client.

ESP-Components
ESP32DLEDController
released

Controls WS2812B type digital LEDs using the RMT peripheral of ESP32.

Depends on the ESP32RMT component.

Used by Dev33 DLED and Dev34 DLEDGrid.

ESP32RMT
released

This is a helper component for using the ESP32's Remote Control module.

Uses the RMT driver from `ESP-IDF` and can be used to control any of the eight channels of the RMT module.

Used by ESP32DLEDController to control WS2812* digital LEDs.

ESP32SimpleOTA
released

This is a helper component to perform OTA updates for ESP32.

Used by Dev34 DLEDGrid.

ESP32Timers
released

This is a helper component for using ESP32's timers.

Used by Dev12, Dev33 and Dev34 DLEDGrid.

Code library
Debouncer
released

Helper class for button debouncing.

This should be platform and compiler agnostic.

Used by Dev12, Dev33 and Dev34 DLEDGrid.

esp32_digitalLEDs
archived, end-of-support

Controls WS2812B type digital LEDs using the RMT peripheral of ESP32.

This implementation uses the RMT driver from ESP-IDF and uses one RMT channel for a LED strip leaving the other channels free, under the control of the RMT driver.
